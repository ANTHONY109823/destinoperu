1. Resumen ejecutivo

Destino Perú es un marketplace de turismo peruano con catálogo público de experiencias, reservas con pago Yape/Plin, programa de fidelidad, y paneles operativos para Super Administrador, administradores de agencia, vendedores y clientes. Los errores críticos del panel Super Admin (categoría no encontrada y pestañas inactivas) están corregidos.

2. Arquitectura y conexiones

2.1 Diagrama lógico

Cliente (navegador) → Blazor WASM → HTTPS JSON + JWT → DestinoPeruAPI → PostgreSQL (Railway). Imágenes vía Cloudinary. Auth y carrito en localStorage.

2.2 Base de datos

La API resuelve la cadena de conexión en este orden:

  1. DATABASE_URL / variantes Railway
  2. ConnectionStrings__DefaultConnection
  3. Variables PGHOST, PGPORT, etc.
  4. appsettings.json (solo desarrollo local)

2.3 Frontend → API

6. Seguridad — evaluación

6.1 Controles implementados

  • Contraseñas con BCrypt (factor 12).
  • Registro público forzado a rol Cliente.
  • Endpoints sensibles con [Authorize(Roles = ...)].
  • Suplantación valida vínculo partner/vendedor.
  • Swagger solo en entorno Development.
  • Actualización de tours valida PartnerId del recurso.

8. Correcciones recientes (mayo 2026)

  • Categoría no encontrada: resolución de ruta ignora ?tab= en la URI.
  • Pestañas Super Admin inactivas: corregido Tab="mainTab" (literal) → @bind-Tab="mainTab".
  • Dashboard Inicio: gráficos donut/barra y navegación a categorías con pestaña.
  • Operaciones Admin: cajas compactas, asientos de bus (total/libres/vendidos).
  • Descargas: manifiesto CSV/PDF y ranking CSV (commit 648994f).
